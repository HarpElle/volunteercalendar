## 1. Root causes

### Blocker 1 — Self-Service zero-assignment draft → "no volunteers" empty state

`src/app/dashboard/schedules/page.tsx` had a binary check:

```tsx
{activeAssignments.length === 0 ? <ScheduleEmptyState .../> : <ScheduleMatrix .../>}
```

For Self-Service, zero assignments is *intentional* — every role is meant to be a claimable slot — but the empty state's `hasServices && !hasVolunteers` branch fired anyway because the page wasn't filtering volunteers against the schedule's `ministry_ids`. Net effect: the wizard promised open slots and the page contradicted it with "selected team has no volunteers."

### Blocker 2 — Ministry Approvals summary tile still showed UID + UTC date

PR #27 fixed `MinistryReviewPanel` (the expanded panel under "Hide Review") but left `ApprovalCountdown` (the per-ministry tile grid above it) untouched. The two surfaces shared no helper — `ApprovalCountdown` rendered ``Approved ${formatDate(...)} by ${approval.approved_by}`` directly, where `formatDate` used the browser TZ and `approved_by` was a raw Firebase UID.

## 2. Files changed (5)

- **`src/components/scheduling/self-service-open-slots.tsx`** *(new)* — Renders a Self-Service schedule's open slots. Computes occurrences from `generateOccurrences(scopedServices, dateStart, dateEnd)`, groups by date, then service, then ministry; each role slot is a red-dashed `1× {role.title} needed` chip matching the matrix's unfilled styling. Shows a small coral summary header with the open-slot count + the same "volunteers claim from My Schedule" copy used in the wizard.

- **`src/components/scheduling/approval-countdown.tsx`** — Accepts `volunteers?: Person[]` and `churchTimezone?: string`. Adds the same `personByUserId` map + `formatApprovedAt` + `resolveApproverName` helpers `MinistryReviewPanel` uses. Approved + Rejected tile copy now mirrors the expanded panel ("Approved {church-local date} by {Person.name}") and adds a tooltip on the approved line.
- **`tests/unit/self-service-generator.test.ts`** *(new)* — locks in the Self-Service generator contract: zero assignments regardless of eligible volunteer count, one `unfilled_role` conflict per slot, centralized/omitted-mode still auto-assigns. Also covers `generateOccurrences` since the UI calls it directly.
- **`tests/unit/approval-metadata.test.ts`** *(new)* — pure-helper coverage for the church-TZ date formatter and UID-to-name resolver: across-DST-day proof (`2026-05-18T03:00:00Z` → "May 17, 2026" CDT vs "May 18, 2026" UTC), UTC fallback when `timezone` is undefined, raw-UID fallback when no Person matches, empty string for null/undefined UID.

## 7. Edge cases worth watching

- **Self-Service view scope respects schedule.ministry_ids.** If the admin scoped the draft to "Worship Team," only Worship services appear in the open-slots view. Org-wide drafts show all services.
- **Open-slot count counts every slot, including roles with `count > 1`.** A service with `Vocalist × 2` renders two chips and contributes 2 to the open-slot summary. Matches the matrix's unfilled counter.
- **Approver UID fallback.** If the approver is no longer a Person in this org (left + removed), the tile shows the raw UID. Consistent with the expanded panel's behavior; can revisit if a real customer hits it.
- **Rejected tiles now also show the rejecting user's name.** That wasn't in scope of the original ask but follows from sharing the helper — previously they only showed the date.
- **TZ formatter swallows malformed IANA strings.** If the church doc somehow has `timezone: "Not/A/TZ"`, the formatter falls back to the browser's locale instead of crashing. Covered by `approval-metadata.test.ts`.
