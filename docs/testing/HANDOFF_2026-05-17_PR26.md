## 1. Root cause

`/api/facility/reservations` has existed for a while but **no frontend ever consumed it**, and **no UI existed to tag a room with `facility_group_id`**. Net effect:

- Every room had `facility_group_id` unset → nothing was actually shared, even though the orgs were facility-group members.
- `/calendar` and `/dashboard/rooms` both fetch own-org-only data; neither merged linked-org rooms or reservations.
- So even with the correct setup-code invite + accept flow (passing in PR #25), the cross-org visibility check had no data to find and no surface to render it.

This was **missing implementation** — the feature was half-built. The backend half existed; the frontend half didn't.

## 2. Design decision (now consistent)

- **Tagging is per-room**, opt-in. Admins explicitly mark which rooms are visible cross-org via a new dropdown in the Edit Room form.
- **Cross-org visibility is on a dedicated calendar** at `/dashboard/rooms/facility/[groupId]` — not merged into the main `/calendar`. Rationale: keeps the own-org admin view clean, and the partner-org reservations are clearly attributed by org+room on the dedicated view.
- **Both orgs see each other.** The dedicated calendar shows rooms + reservations from every OTHER org in the group. To see your OWN rooms in the group, use `/calendar` as usual.

## 9. Edge cases worth watching

- **Tagging is opt-in.** Rooms without `facility_group_id` are NOT visible cross-org, even if both orgs are members of a group. This is intentional — admins explicitly decide which rooms are shared.
- **Own-org rooms don't appear on the shared calendar.** That's by design. To see your own rooms, use `/calendar`. The shared view is purely cross-org.
- **Only one facility group per room.** A room can be tagged with at most one group at a time. If you need a room visible to multiple groups, you'd need multiple facility_group_ids — not supported today. Flag if a real customer needs it.
- **`pending_approval` reservations on shared rooms are visible.** They render with a `Pending` badge in the week-view. Good for capacity planning across orgs.
- **The dropdown only shows ACTIVE facility-group memberships.** If a partner declines or you leave, the option disappears (but already-tagged rooms keep their `facility_group_id` until you edit). Could be a follow-up to auto-clear orphan tags.
