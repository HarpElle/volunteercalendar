## 1. What shipped

### Polish #1 — Cancel session feedback

**Before:** clicking *Cancel session* on the detail page returned to `/dashboard/training-sessions` with the default `Scheduled` filter active → the just-cancelled session was invisible.

**After:** detail page redirects to `/dashboard/training-sessions?status=cancelled`. List page now reads `?status=` query param on mount (validates against the 4 known values, falls back to `scheduled`). The cancelled session is immediately visible in its new home.

### Polish #2 — Mark Complete modal copy

**Before:** *"N attendees will have their {prereq} prerequisite marked complete"* — true for fresh attendees, but the API silently skips anyone already cleared. The final toast then said "0 prerequisite steps auto-completed", reading like a bug.

**After:** *"N attendees will be marked attended. Anyone who still has {prereq} pending will have it auto-completed; volunteers who already cleared the prerequisite are skipped."* The `auto_complete: false` case keeps its distinct copy explaining no journey changes will happen.

### Polish #3 — RSVP Decline → Can't make it

**Before:** *Decline* button on the volunteer RSVP page.

**After:** *Can't make it* — matches the existing copy on the Upcoming-tab row and reads warmer for a church-volunteer context. (Codex's original recommendation.)

### Polish #4 — Self-signup row label

**Before:** *Release* button label on the Upcoming-tab row.

**After:** *Release slot* — the confirm modal's verb already used this phrasing; the row label now matches.

## 7. Edge cases worth watching

- **Polish #1's query-param read happens once on mount.** If you're on `/dashboard/training-sessions?status=scheduled` and a user navigates directly to `?status=cancelled` without a page reload, the filter chips won't change to match. Edge case (no in-app deep-link forces this); not worth a useEffect-on-query-change yet.
- **Polish #2's modal copy distinguishes auto_complete ON vs OFF.** If the session has auto_complete OFF, the modal correctly suppresses the per-attendee distinction (no auto-completion is happening at all).
- **Polish #4 keeps the confirm modal's confirmLabel ("Release slot")** consistent with the row button — Codex's report noted the modal text was already fine.
- **CI smoke pins use the literal apostrophe** (`Can't make it`), not the HTML entity. Next.js's build emits the literal character into the bundle. Verified by local + production grep.
