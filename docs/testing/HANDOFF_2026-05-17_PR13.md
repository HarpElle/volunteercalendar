## 1. Root cause (booking entry)

`RoomBookingForm` — the 5-step wizard the test plan describes — already existed and worked end-to-end. It was only mounted at `/calendar`. The `/dashboard/rooms` list and the room detail page had no entry point, so an admin following the test plan looked in the obvious places and found nothing. The `POST /api/reservations` endpoint was fine.

Per-room equipment and per-room approval were two separate gaps in the same area: the API + types didn't include `requires_approval`, and neither the Add Room modal nor the Edit Room form exposed `equipment` even though the field already existed on the `Room` type.

Display + iCal URLs were copy-only — buttons that wrote the URL straight to the clipboard with no on-screen preview. Codex couldn't inspect or open them.

- `src/app/api/reservations/route.ts` — `(orgWide.require_approval || room.requires_approval)` resolution for both single + recurring paths
- `src/app/dashboard/rooms/page.tsx` — `New Reservation` button, `RoomBookingForm` mount, Add Room modal equipment checkboxes + approval toggle, `Approval` badge on room cards

- **Equipment ⇄ wizard step 3.** The booking wizard's step 3 lists equipment from the *selected room*. If you booked Sanctuary you should see Projector + Sound system as the toggleable options; if you booked Classroom 1 you should see Whiteboard + Tables. The wizard does not show your full global equipment tag list — that's intentional (per-room inventory, not org-wide).
- **Recurring on Pro tier.** Recurring step should be enabled on Pro. If you see "recurring is not available on your tier" or the step is skipped, flag it.
- **Visible URLs.** The Display URL panel now shows the URL inline. The first time you open it on an iPad, the wake-lock indicator should appear. If the URL works in the panel but not in an actual second-screen tab, capture both.
- **Empty equipment_tags case.** If you wipe the global equipment tags first, the Add/Edit Room modal shows a "Add equipment tags in Room Settings first" link instead of checkboxes. Workable but flag if the copy is confusing.
- **`/dashboard/rooms/settings`** still redirects to `/dashboard/org/campuses`. This is **deferred polish** — the redirect target does contain the settings, so the redirect is functional. Flag if you hit any confusion beyond the wording.
