## 1. Root causes

### Blocker 1 — `2026-08-24 → 2026-08-30` Self-Service draft rendered as 3/3 assigned matrix

The schedule doc was correctly stored with `workflow_mode: "self-service"`, but three auto-generated assignments survived from before PR #27 added the `workflow_mode === "self-service"` skip to `generateDraftSchedule`. The page-level conditional `activeAssignments.length === 0 && normalizeWorkflowMode(...) === "self-service"` failed the first check because those stale assignments were still in Firestore.

This is a **data** problem, not a code problem. New self-service drafts created after PR #27 (e.g. `2026-09-07 → 2026-09-13`, `2026-10-05 → 2026-10-11`) have zero assignments and rendered correctly all along — Codex confirmed this.

### Blocker 2 — Nov 9–14 Centralized draft showed role-definition mismatch copy

`ScheduleEmptyState`'s `hasServices = scopedServices.length > 0` only checks whether services *exist* for the scope, not whether any of them *occur* in the date range. For Mon–Sat with a weekly Sunday service, `hasServices` was `true` so the empty state fell into the "Check service role definitions" branch — completely the wrong diagnostic.

## 2. Files changed (3)

- **`src/app/dashboard/schedules/page.tsx`** — `ScheduleEmptyState` adds a `hasOccurrences = generateOccurrences(scopedServices, start, end).length > 0` check and a dedicated "No services occur between {start} and {end}…" branch with a "Manage Services" CTA. The role-definition mismatch branch is now properly gated on having occurrences AND volunteers.
- **`scripts/backfill-self-service-stale-assignments.ts`** *(new)* — idempotent admin script. Finds every self-service draft across every church and deletes assignments with `signup_type === "scheduled"` (auto-generated). Preserves `self_signup` (future volunteer-claim) assignments. Dry-run by default; `--apply` to write. Safe to rerun.
- **`tests/unit/self-service-generator.test.ts`** — new test covers the Nov 9–14 × weekly Sunday case so the no-occurrences regression can't silently reopen.

## 8. Edge cases worth watching

- **The backfill only touches `draft` schedules.** Approved/published self-service schedules (none exist today) would keep their assignments. That's intentional: once a schedule moves past draft, the assignment data is part of the published record and shouldn't be mutated by a backfill.
- **`signup_type` defaults to `"scheduled"` when missing.** The Assignment type makes it required, but older docs might omit it. The script treats missing as scheduled (auto-generated) for safety — anything that should be preserved would have an explicit `self_signup`.
- **The no-occurrence branch reuses `generateOccurrences()`** — the same helper the Self-Service open-slots view uses. So if you ever see a Centralized draft fall into "no occurrences" but a Self-Service draft for the same range shows occurrences (or vice versa), they're using inconsistent logic. They should track together by definition.
- **The Mon–Sat × Sunday-only test** is the precise Codex case; it's also a useful canary against any future scheduler change that broke the weekly-day-of-week walk.
