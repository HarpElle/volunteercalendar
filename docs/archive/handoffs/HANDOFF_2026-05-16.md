## Before you send this to Codex (Jason's deploy checklist)

The fixes are in code on the `claude/infallible-ptolemy-773897` branch (or wherever the merge lands). They are **not yet live** until you:

1. **Commit + push + merge** the branch to `main`
2. **Deploy Firestore rules**: `firebase deploy --only firestore:rules`
3. **Deploy Firestore indexes**: `firebase deploy --only firestore:indexes`
   - Wait until the new `short_links (slug, expires_at)` index status flips to "Enabled" in Firebase Console → Firestore → Indexes (usually <60s for the small `short_links` collection)
4. **Run the backfill script in dry-run mode** to preview the change list:
   ```bash
   npx tsx scripts/backfill-calendar-feed-owners.ts
   ```
5. **Run the backfill in apply mode** (destructive — invalidates every existing feed token):
   ```bash
   npx tsx scripts/backfill-calendar-feed-owners.ts --apply
   ```
   ⚠ Anyone subscribed to a VolunteerCal calendar feed will need to resubscribe with a fresh URL after this. The leaked Codex test token (the one Alex Kim could see in his Account page) gets invalidated as a side effect — that's the point.
6. **Verify in production** with a fast smoke test (5 min):
   - Sign in as a volunteer test account → Account Settings → confirm Calendar Feeds section only shows your own feeds (none if you haven't created any)
   - Sign in as the test admin → create a tiny schedule → confirm `Send for Review` → `Approve Schedule` works in one click on Free
   - Visit `https://volunteercal.com/dashboard` while logged out → confirm redirect to `/login`
   - Visit `https://volunteercal.com/s/bogus`, `https://volunteercal.com/join/nothing`, `https://volunteercal.com/confirm/badtoken` → confirm friendly error pages, no crash screens
   - Submit the landing-page waitlist form → confirm success message (instead of "Failed to save submission")
7. **Decide on Codex's existing test org.** Recommended: have Codex start with a **fresh org** (`TESTER — Codex 2`). The previous `TESTER — Codex` has corrupted state from the eligibility bug (3 volunteers assigned to 12 Worship slots they shouldn't be on) and from any rotated calendar feeds. If you want Codex to continue with the old org, you'll need to manually clean up assignments and ministry memberships first.
