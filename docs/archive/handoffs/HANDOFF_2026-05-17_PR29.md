## 1. Root cause

I queried the Firestore doc for the failing draft (`churches/NQUd09eP1fOD3MWrdpIbHYUXm0z2/schedules/n6IjRExZqOfsAXgnlZI1`, `2026-09-07 → 2026-09-13`). It stores `workflow_mode: "self-service"` exactly — identical to the fresh draft that works. The fresh draft's render path is hit immediately after generation; the existing draft's path is hit via `viewSchedule()` from the list. Both should evaluate `activeSchedule.workflow_mode === "self-service"` to true, so the strict-equality conditional should have matched. It didn't — and I couldn't reproduce it after the fact.

Rather than chase a stale-state ghost, PR #29 makes the entire render path bullet-proof so neither this draft nor any future one can miss:

1. **`normalizeWorkflowMode()` accepts any variant.** Whitespace, underscores, casing — all collapse to the canonical `"self-service"` (or whichever mode applies).
2. **Page-level conditional uses the normalizer.** No more strict-equality miss.
3. **`ScheduleEmptyState` short-circuits to the open-slots view on a self-service schedule** regardless of how the empty state was reached.
4. **`ScheduleEmptyState`'s `hasVolunteers` check was broken for org-wide schedules** — `.some(id => [].includes(id))` is always false, so *any* zero-assignment org-wide draft would have shown "the selected team has no volunteers" (false). Org-wide now counts every active volunteer.

That last item is a separate latent bug that this PR also fixes.

## 7. Edge cases worth watching

- **Workflow modes other than self-service** still pass through `normalizeWorkflowMode` unchanged. Centralized stays centralized, Team-First stays ministry-first, Hybrid stays hybrid.
- **Unknown stored workflow values** now return `null` from the normalizer rather than silently coerce. The conditional `=== "self-service"` will be false, so an unknown value falls into the empty state — same as today.
- **`hasVolunteers` fix is broader than the open-slots fix.** Any old Centralized org-wide draft that was empty for unrelated reasons (e.g. no services match) will no longer falsely accuse the "selected team" of having no volunteers. The empty state's other branches still surface correctly.
- **Tests cover the normalizer, not the JSX.** The page conditional + the empty-state safety net are still indirectly verified via tsc on the file; for true UI coverage you'd need a Playwright suite which isn't in this codebase yet.
