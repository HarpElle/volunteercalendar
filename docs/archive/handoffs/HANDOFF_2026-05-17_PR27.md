## 2. Design decisions worth knowing

- **Self-Service draft starts with zero assignments.** Every role becomes an unfilled slot (`conflicts: type "unfilled_role"`). The wizard's review step now warns about this explicitly. The full open-slot **claim** UI for volunteers is **deferred** — for now schedulers manually fill via Reassign/drag.
- **Availability banner is org-scoped per volunteer.** When admin clicks Send Availability Request, the existing email batch fires *and* a `user_notifications` doc of type `availability_request` is written for every volunteer whose Person has a linked `user_id`. The volunteer sees a coral banner on `/dashboard/my-schedule` with **Submit Availability** (deep-links to `/dashboard/my-availability`) and **Dismiss** (marks the notification read).
- **Trainee toggle uses the existing scheduler permission model.** PATCH `/api/assignments/[id]` rejects anyone below `scheduler`. Matrix chip menu only renders when the active schedule is `draft` or `in_review`.
- **Trainee chip does NOT count toward fill.** Service-level unfilled badge subtracts trainees, so a role with only a shadow assignment still appears as "1× Role needed."
- **Approval badge uses church TZ.** Falls back to UTC if the church doc lacks `timezone`. Approver UID resolves via `Person.user_id` map; falls back to the raw UID only when no Person matches (e.g. the approver is no longer a member).

## 9. Edge cases worth watching

- **Availability banner skips people without a linked user account.** If a Person has no `user_id` (e.g. CSV-imported, never invited), no notification is created — the email still fires but the in-app banner won't surface for them. Expected behavior.
- **Self-Service draft is intentionally empty.** Stats bar will show very low fill rate. That's correct — every slot is supposed to be claimable. Until the volunteer claim UI ships, schedulers fill manually via Reassign.
- **Trainee → Regular flip can over-fill a slot.** If a role with 1 slot already has 1 regular assignment + 1 trainee, flipping the trainee to Regular makes 2 assignments for 1 slot. No backend block; UI will show `0 needed` but the matrix still renders both chips. Worth a note for future work but not a current blocker.
- **Approver name resolution falls back to UID.** If the approver is no longer a member (left the org), the badge will show the raw UID again. Acceptable in a rare deletion case.
- **Wizard self-service note is informational only.** Selecting Self-Service still requires the admin to make sense of the empty draft. We don't block generation.

3. **Availability banner should auto-clear if the campaign deadline passes.** Currently it only clears on dismiss/read. Add an `expires_at`-based filter on render.
4. **Approval badge for `rejected` status** still doesn't show who rejected or when. Same fix would apply but UX hasn't shown that as a need yet.
