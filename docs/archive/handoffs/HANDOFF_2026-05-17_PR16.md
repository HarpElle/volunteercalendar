## 8. Edge cases worth watching

- **Edit scope behavior.** The PUT endpoint, when called with `edit_scope: "from_date"`, batches an update across every reservation in the same `recurrence_group_id` where `date >= existing.date`. The selected occurrence IS included. Flag if it's not.
- **Date range chip persistence.** The selected chip resets when you navigate away. Flag if you'd prefer it sticky.
- **Cancel UI.** The DELETE endpoint sets `status: "cancelled"` rather than hard-deleting. After cancel, the row disappears from the list because the GET filters to `confirmed` + `pending_approval` by default. The cancelled doc is still in Firestore (audit trail). No "Show cancelled" toggle yet — flag if you want one.
- **Edit-only-time vs. edit-everything.** The modal only exposes start/end time editing. Title, description, equipment, recurrence can't be edited inline yet — they'd require re-opening the full booking wizard. Flag if you hit a Phase 5 step that needs those.
- **Organizer name source.** The fix reads `users/{uid}.display_name` → `name` → `email`. If you're signed in as an admin whose user doc has no `display_name`, it'll fall through to email. Auth `displayName` is not consulted on the server (since the server doesn't have it on the request).
