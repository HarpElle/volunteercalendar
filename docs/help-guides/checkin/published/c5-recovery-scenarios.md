# C5 — `c5-recovery-scenarios`

## Recovery scenarios at pickup

**Audience:** Admin handling the situations that don't fit the normal flow.

**Outcome:** You can release a child safely when the code or phone is missing, or when a different guardian arrives.

---

### The parent doesn't have their security code

Look up the child's session in the admin tools — you can surface the security code from there and confirm it against the child's name tag.

[SCREENSHOT: c5-session-lookup-code]

---

### The parent can't get their text

If the code never reached the parent's phone, verify their identity another way — a driver's license, for instance — and release the child once you're satisfied.

---

### A second guardian arrives instead of the one who checked in

The authorized pickup list governs. If the person arriving is on that household's authorized list, they can collect the child. If they aren't, don't release until you've confirmed they're authorized.

[SCREENSHOT: c5-authorized-pickup-list]
